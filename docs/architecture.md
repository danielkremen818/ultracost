# Architecture

ultracost is **one shared core** (`src/`) with a single source of truth (`policy.json`),
exposed through **two delivery surfaces** — a Claude Code **plugin** (primary) and an
**npm CLI** (secondary) — plus a **runtime verification layer** (the static guard and a
`PreToolUse` cost gate) and a **closed loop** that reconciles and calibrates from real
runs. Everything compiles from the same policy, so the routing rules can never drift
between surfaces.

## The shared core (`src/`)

All zero-dependency Node, grouped by capability. `index.js` is the package export
surface; `log.js`/`render.js` are the styling kit; everything else does one job.

| Module | Responsibility |
|---|---|
| `policy.js` | Load, validate, and normalize `policy.json`; resolve a model alias to a tier. |
| `rules.js` | Compile the policy into the deterministic CLAUDE.md routing block (the single rule text both surfaces use). |
| `lexer.js` | A hand-rolled, zero-dependency JS tokenizer (no full parse) that locates real `agent()` call sites and reads option literals. |
| `guard.js` | Static analysis over the token stream: counts every stage and flags `UC001`–`UC008`; also `audit`, `--fix`, and the per-stage `stageList`. |
| `classify.js` | Deterministic, offline prompt→tier scorer that backs the `UC006`/`UC008` advisories and `explain`'s "reads-like" column. |
| `estimate.js` | Static cost estimate: agent count, model mix, and tiered-vs-all-opus cost (effort-weighted). |
| `pricing.js` | Parse + refresh the official rate table (the one optional outbound request, only on `pricing refresh`). |
| `transcript.js` | Read local Claude Code session transcripts offline; attribute tokens per dynamic-workflow stage; dedup on `message.id`+`requestId`. |
| `cost.js` | Price real transcript usage with cache-aware multipliers. |
| `loop.js` | The closed loop: `reconcile`, `calibrate` (a self-tuning token prior), and the persisted savings `ledger`. |
| `detect.js` | Report how ultracost is delivered (`plugin`/`cli`/`both`/`none`) and surface the bypass-mode caveat, so `status`/`doctor`/`init` never misreport or double-install. |
| `install.js` | The CLI install/uninstall path: writes the policy, the CLAUDE.md block, and the `SessionStart` hook (defensive about invalid JSON; fully reversible). |
| `paths.js` | All filesystem locations (honors `CLAUDE_CONFIG_DIR`) and `safePath()`, the single path choke point. |
| `render.js` | Truecolor/256/16 terminal kit (panels, tables, bars, sparklines, gradients) with ANSI-aware width; honors `NO_COLOR`/`FORCE_COLOR`. |

## The two surfaces

| | Plugin (primary) | npm CLI (secondary) |
|---|---|---|
| **Install** | `/plugin marketplace add danielkremen818/ultracost` → `/plugin install ultracost@ultracost` | `npx ultracost init` |
| **Routing guidance** | `SessionStart` hook injects the policy as context (no file mutation); a routing skill ships alongside for explicit reference | the block is injected into `~/.claude/CLAUDE.md` |
| **Guard** | `/ultracost:check` (+ a slash command for every verb) | `ultracost check` / `ultracost audit` |
| **Policy injection** | `hooks/hooks.json` → `node "${CLAUDE_PLUGIN_ROOT}/templates/hooks/reinject.mjs"` | `node "<config>/ultracost/reinject.mjs"`, registered in `settings.json` |
| **Cost gate** | `hooks/hooks.json` → `PreToolUse` (matcher `Workflow`) runs `workflow-gate.mjs` by default | run `ultracost estimate <script>` by hand, or wire the same hook |
| **Closed-loop autorun** | `hooks/hooks.json` → `Stop` runs `loop-autorun.mjs` — reconcile + calibrate + ledger after a workflow finishes | `ultracost init` registers the same `Stop` hook in `settings.json` |
| **Best for** | day-to-day use, sharing, marketplace discovery | CI, scripting, the CLAUDE.md-injection path |

Both surfaces are compiled from the same policy. The skill (`skills/ultracost/SKILL.md`)
and the slash commands (`commands/*.md`) are **generated** from `src/rules.js` and one
table — `scripts/generate-surfaces.mjs` and `scripts/generate-commands.mjs` — and
`tests/surfaces.test.js` fails if either drifts.

## 1. Policy (`src/policy.js`)

Loads, validates, and normalizes `policy.json`. Resolution order:

1. explicit path argument
2. installed `<config>/ultracost/policy.json`
3. bundled `templates/policy.default.json`

`<config>` is `~/.claude` by default, or `$CLAUDE_CONFIG_DIR` when set — every ultracost
path hangs off it, so a relocated Claude Code config is honored automatically. Validation
rejects a `default` tier that isn't defined and any tier whose `model` is in `neverUse`, so
a broken policy can never be installed.

## 2. Rules + install (`src/rules.js`, `src/install.js`)

`compileRules(policy)` renders a deterministic Markdown block (between
`<!-- ultracost:start -->` / `<!-- ultracost:end -->`) so the prose rules can never drift
from the policy data. The same compiled block backs the plugin skill, the CLAUDE.md block,
and the SessionStart hook's injected text.

On the **CLI** path, `install()` writes the policy to `<config>/ultracost/policy.json`,
injects/updates the block in `<config>/CLAUDE.md` (the **canonical** user-global memory —
`~/CLAUDE.md` is not loaded reliably and is never used), copies the re-inject hook, and
registers it for `SessionStart` (matcher `startup|resume|clear|compact`) in `settings.json`.
All `settings.json` access is defensive: a missing file is created; an invalid file is
reported, never overwritten. On the **plugin** path nothing in your config is mutated — the
plugin loader wires up the bundled skill, commands, and hooks.

## 3. The Workflow Guard (`src/guard.js`, `src/lexer.js`, `src/classify.js`)

A heuristic, dependency-free static analyzer for the JavaScript workflow scripts Claude
Code authors. It runs on a hand-rolled tokenizer (`lexer.js`) — not regex — so it is robust
to template literals, spreads, optional-call `agent?.()`, and dynamic values, and an
`agent(` inside a prompt string or comment is treated as prose, never a call.

It classifies each stage: missing options (`UC001`), missing model (`UC002`), banned model
(`UC003`), `inherit` (`UC004`), dynamic/variable options (`UC005`), and three advisory
checks — wrong tier for the work (`UC006`), effort over the model's cap (`UC007`), and an
`alwaysOpus` role pinned off-opus (`UC008`). `UC001`–`UC004` are errors (non-zero exit);
`UC005`–`UC008` are warnings. `audit [dir]` reuses the same analysis over
`<dir>/**/workflows/scripts/*.js` for the headline pin-coverage stat; `--fix` inserts the
default tier's model into the unambiguous cases (`UC001`/`UC002`) only, back-to-front so
edits don't shift later offsets.

## 4. The runtime verification layer

Two hooks turn the policy from advice into enforcement:

- **`SessionStart` (`templates/hooks/reinject.mjs`).** A cross-platform node script (no
  bash/`jq`) that emits the routing policy as `additionalContext` on every `SessionStart`
  source, so the policy is present *before the first prompt* — not dependent on the model
  choosing to open a skill. It compiles the text from `rules.js` at runtime, so it cannot
  drift from `policy.json`.
- **`PreToolUse` cost gate (`templates/hooks/workflow-gate.mjs`).** Registered on the
  `Workflow` tool, **on by default**. It reads the drafted script, runs the static guard
  **and** the (calibrated) estimate, enforces the policy budget caps, and returns a
  permission decision with an aligned cost table — so an accidental all-Opus fan-out or an
  over-budget launch can't slip through. It is mode-aware: it asks (with the estimate) in
  `default`/`acceptEdits`/`auto`, and hard-denies a problem workflow in
  `bypassPermissions`/`dontAsk` where an ask wouldn't pause. `ULTRACOST_GATE` overrides it
  (`strict`/`ask`/`off`). Full detail in [ESTIMATES.md](./ESTIMATES.md).

## 5. The closed loop (`transcript.js`, `cost.js`, `loop.js`)

After a run, ultracost reads its own results back and tunes itself — all offline, from
local transcripts:

- **`transcript.js`** parses Claude Code's `*.jsonl` session files and attributes tokens
  **per dynamic-workflow stage** via `subagents/workflows/wf_*/agent-*.jsonl` + `journal.jsonl`
  (separation is by path, never `sessionId`).
- **`cost.js`** prices that usage with cache-aware multipliers.
- **`loop.js`** turns it into `reconcile` (estimate vs actual per stage), `calibrate` (an
  outlier-filtered token prior that `estimate`/`explain`/`simulate`/the gate then use
  automatically), and a persisted `ledger` (cumulative savings vs all-opus, idempotent per
  run). `calibrate` feeding back into the estimator is what closes the loop.

## Why this shape

- **Data over prose.** The policy is machine-checkable and user-editable; the rule text is
  generated for every surface from the same source.
- **Defense in depth.** The `SessionStart` hook *injects* the policy as always-on context;
  the guard and the `PreToolUse` gate *verify* the output. The model can ignore guidance; it
  cannot ignore a failing check or a hard-denied launch.
- **Narrow on purpose.** ultracost only routes the workflow/ultracode path, so it stays
  composable with a general router if you run one.

## Limitations

- The guard is a heuristic scanner, not a full JS parser. The tokenizer removes the common
  false positives (prompt text containing `agent(`), but deeply unusual constructs could be
  misclassified; dynamic/variable options are reported as warnings (`UC005`), never guessed.
- `--fix` only handles the unambiguous cases (`UC001`/`UC002`); banned models and `inherit`
  are reported for you to resolve, never silently rewritten.
- Estimates are relative (tiered vs all-opus), not a bill; fan-out is a range; the gate's
  interactive pause needs a TUI (a `deny` is honored everywhere). See
  [ESTIMATES.md](./ESTIMATES.md).
